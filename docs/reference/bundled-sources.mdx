Coral ships with bundled source specs for common systems. If the system you need is not listed here, you can [write a custom source](/guides/write-a-custom-source) for any HTTP API or local dataset.

To see the bundled sources available in your build:

```shellscript
coral source discover
```

To install one:

```shellscript
coral source add github
```

To validate an installed source:

```shellscript
coral source test github
```

## Setup expectations

Most bundled sources fall into one of two categories.

### API sources

These usually require:

- one or more API tokens or credentials
- sometimes additional variables such as base URLs, sites, or organization IDs

When you run `coral source add <NAME>`, Coral prompts for the required values interactively.

### File-backed sources

These point at local files or object-store locations instead of calling an API.

Examples:

- `otel` reads Parquet exports
- custom `jsonl` or `parquet` source specs can point at local files

## Querying sources safely

Use [Quickstart](/getting-started/quickstart) for the basic flow, then use [System catalog](/concepts/system-catalog) when you need to inspect schemas, columns, descriptions, and required filters before writing larger queries.

## Don't see what you need?

The bundled set is growing. If your system is not listed, [write a custom source](/guides/write-a-custom-source) — source specs can connect to any HTTP API or read local files in JSONL or Parquet format.
