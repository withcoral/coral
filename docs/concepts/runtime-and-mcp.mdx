## Runtime

The Coral runtime is the local engine that:

- loads installed sources
- compiles them into queryable backends
- exposes them through SQL

The CLI and MCP server both use the same underlying runtime.

For normal CLI usage, Coral bootstraps an internal local gRPC server under the hood through `coral-client`. You do not need to manage a separate daemon before running commands such as `coral sql`.

## MCP

MCP is the protocol Coral uses to expose its local runtime to coding agents.

Today Coral ships an MCP stdio server:

```shellscript
coral mcp-stdio
```

That lets an MCP-capable client use Coral's SQL and schema discovery surface without needing a separate one-off tool for each source.

Today the MCP surface is intentionally small:

- tools: `sql`, `list_tables`
- resources: `coral://guide`, `coral://tables`

## Local-first

Coral is local-first today.

That means:

- install and configure sources locally
- query them locally
- expose that same local runtime to agents over MCP

The current repository does not include a hosted control plane or shared team workspace product.
