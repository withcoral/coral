Coral is a CLI-first product. Beyond the basic querying shown in the [Quickstart](/getting-started/quickstart), the CLI offers flexible output formats for scripting and automation.

## Output formats

By default, Coral formats results as a human-readable table. You can change this using the `--format` flag:

- `table` (default)
- `json`

### Return JSON

```shellscript
coral sql --format json "SELECT * FROM coral.tables LIMIT 10"
```

## Scripting and piping

Because Coral can output JSON, you can pipe its results directly into tools like `jq` for further processing:

```shellscript
coral sql --format json "SELECT schema_name, table_name FROM coral.tables LIMIT 10" | jq '.[].table_name'
```

## Multiline queries

For larger queries, you can format your SQL across multiple lines in your shell:

```shellscript
coral sql "
  SELECT name, draft, published_at
  FROM github.releases
  WHERE owner = 'withcoral' AND repo = 'coral'
  ORDER BY published_at DESC
  LIMIT 10
"
```

## Local runtime model

The default CLI bootstraps an internal local gRPC server through `coral-client`, connects to it, and keeps it alive for the lifetime of the client handle. You do not need to start a separate long-running daemon before running normal `coral sql` commands.
