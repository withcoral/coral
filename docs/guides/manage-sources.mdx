---
title: "Manage installed sources"
description: "Understand how to list, test, update, and remove sources in your local workspace."
---

Once you have added a source via the [Quickstart](/getting-started/quickstart), you may need to manage your installed sources, update their configuration, or remove them.

## List installed sources

To see what is currently active in your local workspace:

```shellscript
coral source list
```

## Validate a source

If a source is failing or you suspect credentials have expired, run:

```shellscript
coral source test <NAME>
```

This command tests connectivity and re-fetches the schema. It is the safest thing to do when troubleshooting an integration.

## Remove a source

When you no longer need a source:

```shellscript
coral source remove <NAME>
```

## Reconfigure a source

If you need to update a token or variable, simply re-add the source. Coral will prompt you to overwrite the existing configuration and will ask for the required inputs.

For bundled sources:

```shellscript
coral source add <NAME>
```

For custom YAML sources:

```shellscript
coral source add --file <PATH>
```
